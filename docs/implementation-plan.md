# Sloth Cloud 一期实施计划

## 阶段 A: 文档与骨架

- 完成品牌定位、页面规划、BFF contract、上游缺口清单
- 建立 `apps/web` 与 `apps/api`
- 建立 mock/live 双模式

## 阶段 B: 原型页面

- 首页
- 商店/分类页
- 产品详情/购买配置页
- 客户服务详情页

交付标准：

- 可运行
- 可切换主题
- 可切换中英
- 能通过 BFF 获取数据

## 阶段 C: Paymenter 真接口接入

- 打通商品分类读取
- 打通产品详情读取
- 打通服务详情读取
- 对缺失字段做 BFF 级降级与标注

## 阶段 D: 二期准备

- 登录态与用户会话
- 订单提交与支付流程
- 面板服务动作聚合
- 工单、账单、通知中心
- 面板真实控制能力

## 我建议补充的规划点

- 定义品牌设计 token，避免页面越做越散
- 尽早确定 URL 规范，避免路由后改
- 服务详情页先把“状态、网络、账单、操作”分区定下来
- BFF 要从第一天开始做日志与错误归一化
- 给每个真实字段标记来源，后续对接 panel 时更好替换
